🚀 HR AI 產品化深度分析

企業 AI 導入架構 × 獵頭模組商業化

❓ 問題 1：企業 HR AI 導入架構設計

「選育用留要用一個系統嗎？還是分開？」

選項 A：All-in-One（單一系統）

模式：一套系統涵蓋選育用留全部功能

員工生命週期平台
├─ 選（招聘）
├─ 育（培訓發展）
├─ 用（人崗配置）
└─ 留（留任預防）
✅ 優點：
  • 數據統一（候選人 → 員工檔案無縫銜接）
  • 企業最愛（一個供應商 = 簡單）
  • 市場上有成熟方案（Workday、SuccessFactors）
  • 粘性高（換系統成本大）
❌ 缺點：
  • 建置難度高（4 個環節都要做好）
  • 實施週期長（3-6 個月以上）
  • 初期投資大（$500K-2M）
  • 決策複雜（要過 HR + IT + CFO）

選項 B：模組化方案（Modular）

模式：分成 4 個獨立模組，可單獨購買或組合

模組 1：招聘模組（選）— $3,000/月
模組 2：培訓模組（育）— $2,000/月
模組 3：人崗配置（用）— $2,500/月
模組 4：留任預警（留）— $2,000/月
✅ 優點：
  • 快速上市（每個模組獨立開發）
  • 靈活銷售（先買「選」，再加「留」）
  • 降低購買門檻（$3K 而非 $10K）
  • 市場反饋快
❌ 缺點：
  • 數據不夠統一（模組間割裂）
  • 組合銷售複雜
  • 容易被替換（競爭力分散）

🎯 建議方案：混合模式（推薦）

先做模組化，逐步升級到系統化

Phase 1（現在）：
只做「招聘模組」$3-5K/月
集中火力做到最好

Phase 2（6 個月）：
加入「留任預警模組」$7-8K/月
原因：招聘 + 留任 = 完整成本 story

Phase 3（12 個月）：
整合成「完整 HR AI 平台」

❓ 問題 2：獵頭模組產品化

「爬蟲 → 匯人選 → 分析履歷，如何對外販售？有什麼問題？」

你現在有什麼

爬蟲找人選 → 匯入履歷池 → AI 分析履歷 → 推薦排名

已驗證：50+ 候選人的實際案例

市場競爭對手

產品 定位 價格 特點
Huntly 被動招聘 $5-15K/月 加強版 LinkedIn
HireEZ AI 人才尋訪 $50-100K/月 企業級，很貴
SeekOut 全球人才庫 $100K+/年 GitHub + LinkedIn
國內 Breezy HR 招聘管理 2-5K/月 簡單，整合度低

你的競爭優勢 ✅

  • 爬蟲技術：國內企業多數沒做，很差別化
  • 本地化：GitHub 中文技能推論
  • 成本低：$3K/月 vs HireEZ $50K/月（10 倍差異）
  • 已驗證：50+ 候選人的實際案例

❌ 不夠好的 5 個部分（要馬上修）

1️⃣ 候選人品質

問題： 爬蟲找到的人不一定有意轉職
- GitHub 上的人可能已在大公司，根本不看邀請
- LinkedIn 數據不完整（很多人沒更新）

要修：
  • 加入「意圖信號」分析
    • 最近更新 GitHub 代碼 = 活躍度高
    • 有找工作相關代碼 = 有意圖
    • LinkedIn 最近改頭像 = 準備轉職
  • 加入「黑名單機制」
    • 已被聯繫過的人不再推薦
    • 已拒絕的人 3 個月內不推薦
  • 人工驗證層（Top 30% 候選人要人工確認）

2️⃣ 匹配精準度

問題： AI 配對分數 0 分（技能映射不準）

要修：
  • 建立「技能本體論」（200+ 常見技能的同義詞）
    • Python = Py, Python3, py
    • 深度學習 = Deep Learning = DL = 神經網路
  • 引入「業務術語」層（拆解複合要求）
  • A/B 測試匹配模型（用實際回應率調整權重）

3️⃣ 郵件轉化率

問題： 自動生成的邀請信被當垃圾郵件

要修：
  • 個性化深度（提到具體 repo、技術棧）
  • 根據級別差異化
    • Junior：教學和成長機會
    • Senior：帶團隊、決策權、高薪
    • Lead：CEO 願意聊願景
  • 測試郵件主題行（A/B 測試）

4️⃣ 後續跟進機制

問題： 很多候選人第一次不回應

要修：
  • 多層次跟進
    • 第 1 天：初始邀請
    • 第 5 天：温和提醒
    • 第 10 天：換角度
    • 第 20 天：最後一次
    • 30 天後：標記無反應
  • 優先順序分級（不是所有人都跟進 20 天）

5️⃣ 數據隱私 & 法律

問題： 爬蟲抓數據，一旦出事會很麻煩

要修：
  • 明確法律合規
    • 只用「公開」GitHub 數據（合法）
    • LinkedIn 數據可能違 ToS（建議放棄爬蟲）
  • 隱私政策清晰
  • 給候選人「取消訂閱」選項
  • 企業合約中有免責條款

🚧 可能遇到的外部阻礙

1️⃣ 技術層面

GitHub API 限制
  • 免費版：60 requests/hour（太少）
  • PAT：5000/hour（需要認證）
  • 成本：$200-500/月（API 代理層）

LinkedIn 的反爬蟲
  • 2025-2026 越來越嚴
  • 建議放棄 LinkedIn 爬蟲

2️⃣ 市場層面

大廠競爭 (LinkedIn、Indeed 都在做 AI 匹配)

解決方案： 不和他們正面競爭，反而「互補」
  • 你的系統是 LinkedIn Recruiter 的「助手」
  • 幫企業省 LinkedIn 訂閱成本

3️⃣ 客戶層面

企業的技術排斥
解決：免費試用 2 週 + 人工確認

企業的合規顧慮
解決：明確寫在合約 + 提供法律保障

4️⃣ 監管層面

個人信息保護法
  • 中國：2021 年已實施
  • 歐洲：GDPR（罰款最高營收 4%）

策略： 主要服務台灣、新加坡（法律相對寬鬆）

5️⃣ 銷售層面

冷啟動困難
需要 3-5 個「case study」才能說服企業

解決方案： 免費試用 5-10 個企業
他們成功後自然會轉介紹

6️⃣ 產品層面

爬蟲穩定性
  • GitHub 改 API 結構 → 爬蟲 break
  • 候選人刪除帳號 → 數據過期

解決： 監控系統 + SLA 99% + 備用數據源
