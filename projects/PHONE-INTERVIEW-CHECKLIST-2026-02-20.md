## 🎤 電話腳本（5 分鐘版）

### 開場（30 秒）
```
「Hi [名字]，我是 Jacky。我在開發一個招聘自動化系統，
想快速了解一下你們現在的招聘流程，大概 5 分鐘可以嗎？」

// 等他們說「可以」
```

### 問題 1（1 分鐘）
```
「你們現在用什麼系統招聘的？」

期望答案：「104」、「1111」、「LinkedIn」、「人脈」

追問：「有沒有用過 ATS（自動篩選系統）？」
```

### 問題 2（1 分鐘）
```
「每個月大概要篩選多少份履歷？」

期望答案：「50-200 份」

追問：「篩選一份履歷要花多久？」
```

### 問題 3（1 分鐘）
```
「現在系統有沒有什麼問題？」

期望他們說：
- 「履歷品質差」
- 「手工篩選太費時」
- 「104 功能不夠」
- 「沒有 AI 幫忙篩」
```

### 問題 4（1 分鐘）
```
「如果有個系統能自動篩選 + 找到真正有技能的人，
你願意每月付多少？$29、$59、$99、還是 $150？」

// 聽他的反應和數字
```

### 問題 5（30 秒）
```
「如果系統只需要 1 週部署，能立即用，
你們會考慮試用嗎？」

期望答案：「可以」或「需要報老闆」
```

### 結尾（30 秒）
```
「很感謝你的時間。如果我們做出來，可以聯繫你試用嗎？」

// 拿他的 Email / Line ID
```

## 🎯 聽完 3 通後，你會知道

### ✅ 如果他們都說...
| 答案 | 意思 |
|------|------|
| 「現在用 104，沒人幫篩」 | 痛點真實，市場確實存在 |
| 「願意付 $59-$99」 | 定價可行 |
| 「會試用」 | 好生意 |
| 「下週可以試」 | 決策不慢 |

### ❌ 如果他們都說...
| 答案 | 意思 |
|------|------|
| 「已經有系統了」 | 你的假設錯了 |
| 「不願意付錢」 | 免費文化太強 |
| 「要報老闆決策 3 個月」 | 企業市場決策慢（預期中） |

## ⚡ 快速提示

### 什麼時候打？
- 工作日 10:00-11:00 AM（HR 最不忙）
- 或下午 3:00-4:00 PM（有空聊天）
- 不要中午或 17:00+（他們忙）

### 怎麼說才有人願意聊？
```
❌ 錯誤：「我想兜售我的招聘系統」
✅ 正確：「我在做市場調研，想聽聽你們的招聘痛點」

❌ 錯誤：「能幫你們省 50% 成本」
✅ 正確：「想了解一下目前的流程」
```

### 被拒絕怎麼辦？
```
「沒關係，如果以後有需要歡迎聯繫」
// 換下一個人
```

## 📝 電話後的動作

**每通電話結束後，馬上記**：
1. 他們用什麼系統
2. 最大痛點是什麼
3. 願意付多少
4. 一句最有趣的話（可能啟發你定位）

**別等 3 通都聊完，邊聊邊調整問題。**
