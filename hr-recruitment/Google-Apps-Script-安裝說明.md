## 🎯 使用方式

### 方法 A：修改資料後執行（推薦）

1. **修改候選人資料**
   - 打開 Apps Script 編輯器
   - 找到 `const CANDIDATE_DATA = {`
   - 修改裡面的資料（姓名、公司、技能等）

2. **執行生成**
   - 點擊 **▶️ 執行** → 選擇 `createResumeFromData`
   - 等待 5-10 秒
   - 會彈出提示訊息，顯示新文件連結

3. **完成**
   - 新的履歷文件已自動生成
   - 點擊連結即可查看

### 方法 B：使用自訂選單（更方便）

1. 回到模板文件
2. 重新整理頁面（F5）
3. 會看到新的選單：**Step1ne 履歷工具**
4. 點擊 **Step1ne 履歷工具** → **📝 生成匿名履歷**
5. 完成！

## 📋 每次使用前的準備

### 修改候選人資料

打開 Apps Script，找到這段程式碼並修改：

```javascript
const CANDIDATE_DATA = {
  // 基本資料
  推薦日期: "2026-02-13",  // ← 改這裡
  企業: "創樂科技有限公司",  // ← 改這裡
  職位: "Product Manager",  // ← 改這裡
  人選代號: "PM-2026-001",  // ← 改這裡
  
  // Personal Particulars
  中文姓名: "⚪⚪⚪",
  英文姓名: "⚪⚪⚪",
  出生年: "198X",  // ← 改這裡
  年齡: "34",  // ← 改這裡
  
  // ... 其他資料
};
```

### 需要修改的主要欄位：
- ✅ 推薦日期
- ✅ 企業名稱
- ✅ 職位名稱
- ✅ 人選代號
- ✅ 出生年、年齡
- ✅ 學校、科系、畢業年月
- ✅ Summary（個人特質）
- ✅ Skills（技能）
- ✅ 工作經歷（公司1、公司2、公司3...）
- ✅ 薪資資料

## 🔧 進階功能

### 新增更多工作經歷

如果候選人有 4 個以上的工作經歷，在 `CANDIDATE_DATA` 中新增：

```javascript
  // 工作經歷 4
  公司4: "某某公司",
  任職期間4: "2020年1月~2022年12月",
  職稱4: "XXX",
  工作內容4: `1. ...
2. ...`,
  成就專案4: "...",
  管理人數4: "...",
  離職原因4: "...",
```

然後在 `replaceAllText` 函數中新增對應的替換規則。

## ⚠️ 注意事項

1. **模板不會被修改**
   - 腳本會自動複製模板
   - 生成的新文件會儲存在雲端硬碟根目錄

2. **檔案命名規則**
   ```
   [人選代號]-[企業名稱]-[推薦日期]
   例：PM-2026-001-創樂科技有限公司-2026-02-13
   ```

3. **資料檢查**
   - 每次執行前請確認 CANDIDATE_DATA 已更新
   - 特別注意：公司名稱、職位、技能等

4. **授權問題**
   - 第一次執行需要授權
   - 如果遇到「不安全」警告，點擊「進階」→「前往」即可

## 🐛 常見問題

### Q1：執行後沒有反應？
A：檢查右上角是否有彈出視窗被封鎖，允許彈出視窗即可。

### Q2：找不到生成的文件？
A：在雲端硬碟根目錄搜尋文件名稱（例：PM-2026-001）

### Q3：想修改模板格式？
A：直接修改原始模板文件即可，腳本會複製最新版本。

### Q4：可以批量生成嗎？
A：可以！修改腳本，用陣列儲存多個候選人資料，迴圈執行即可。
