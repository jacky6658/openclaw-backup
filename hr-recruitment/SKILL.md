# HR 履歷進件處理技能

**版本**：1.0  
**建立日期**：2026-02-10  
**適用對象**：所有參與 HR 招募的 AI Bot

---

## 🎯 技能目的

此技能定義 HR 履歷進件的標準處理流程與通知格式，確保所有 Bot 以統一規範運作。

---

## 📋 履歷進件通知標準格式

### ⚠️ 強制規範（所有 Bot 必須遵循）

當有新履歷進件時，**必須**按照以下格式發送通知到 Telegram 群組：

```
📋 #履歷進件

👤 **姓名**：[應徵者姓名]
📧 **Email**：[Email 地址]
📞 **電話**：[聯絡電話]
💼 **應徵職位**：[職位名稱]
👔 **負責顧問**：[顧問名稱] ⚠️ **必填欄位**

---

⭐ **評分**：[0-100 分]

**優勢**：
- [優勢項目 1]
- [優勢項目 2]
- [優勢項目 3]

**待確認**：
- [待確認事項 1]
- [待確認事項 2]

---

🔗 [查看完整履歷]([檔案連結])

**建議動作**：✅ 邀約面試 / ⏸️ 待觀察 / ❌ 不適合
```

---

## 📌 欄位說明

| 欄位 | 說明 | 必填 |
|------|------|------|
| 姓名 | 應徵者姓名 | ✅ |
| Email | 聯絡信箱 | ✅ |
| 電話 | 聯絡電話 | ✅ |
| 應徵職位 | 投遞的職位 | ✅ |
| **負責顧問** | **處理此履歷的顧問名稱** | ✅ **必填** |
| 評分 | 0-100 分的智能評分 | ✅ |
| 優勢 | 應徵者的優勢（2-5 項） | 建議填寫 |
| 待確認 | 需要進一步確認的事項 | 選填 |
| 查看完整履歷 | 履歷檔案的連結或路徑 | ✅ |

---

## 🤖 Bot 初始化設定

### 新 Bot 加入時必須設定：

1. **在 TOOLS.md 或 Bot 設定檔中記錄「負責顧問名稱」**
   
   範例（TOOLS.md）：
   ```markdown
   ### HR 履歷進件設定
   - 負責顧問：王小明
   - 負責職位：後端工程師、前端工程師
   ```

2. **若無法取得負責顧問資訊**
   - 填寫「待指派」
   - 通知管理員補充設定

## ✅ 範例

### 正確範例 ✅

```
📋 #履歷進件

👤 **姓名**：張大明
📧 **Email**：zhang@example.com
📞 **電話**：0987-654-321
💼 **應徵職位**：全端工程師
👔 **負責顧問**：林小華

---

⭐ **評分**：92 / 100

**優勢**：
- 5 年全端開發經驗
- 精通 React + Node.js
- 有 AWS 架構經驗

**待確認**：
- 期望薪資範圍

---

🔗 [查看完整履歷](https://example.com/resume.pdf)

**建議動作**：✅ 邀約面試
```

### 錯誤範例 ❌

```
📋 #履歷進件

姓名：張大明
Email：zhang@example.com
（❌ 缺少「負責顧問」欄位）
```
