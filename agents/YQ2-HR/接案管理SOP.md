## 二、案件管理系統

### 2.1 系統資訊

```
後端 URL：https://aijobcasebackend.zeabur.app

API 端點：
├── POST /api/ai/import     → 新增案件
├── GET  /api/ai/leads      → 查詢案件
├── POST /api/ai/update     → 更新案件
├── POST /api/ai/delete     → 刪除案件
├── POST /api/ai/progress   → 更新進度
├── POST /api/ai/cost       → 記錄成本
├── POST /api/ai/profit     → 記錄利潤
└── POST /api/ai/attachment → 上傳附件
```

### 2.2 案件狀態定義

| 狀態 | 說明 | 下一步動作 |
|------|------|------------|
| `待篩選` | 新進來的案子 | 評估是否值得跟進 |
| `報價中` | 已發報價單 | 追蹤客戶回覆 |
| `已接洽` | 客戶有回應 | 深入了解需求 |
| `執行中` | 專案進行中 | 追蹤進度、更新狀態 |
| `已完成` | 專案交付完成 | 確認收款、記錄利潤 |
| `取消` | 案子沒成 | 記錄原因 |
| `婉拒/無法聯繫` | 放棄跟進 | 歸檔 |

### 2.3 聯繫狀態定義

| 狀態 | 說明 |
|------|------|
| `未回覆` | 已聯繫但客戶尚未回覆 |
| `已回覆` | 客戶有回覆 |
| `持續溝通中` | 雙方往來中 |

## 三、每日例行任務

### 3.1 每日早上 09:00 - 案件檢查

**執行步驟**：

1. **查詢所有案件**
   ```
   GET /api/ai/leads
   ```

2. **篩選需要關注的案件**
   - 狀態為「報價中」超過 3 天未回覆
   - 狀態為「已接洽」超過 7 天未進展
   - 狀態為「執行中」有即將到期的 deadline

3. **產出今日待辦通知**
   ```
   📋 今日待辦（YYYY-MM-DD）

   【需跟進】
   • [case_code] 客戶名 - 狀態 - 天數
   
   【Deadline】
   • [case_code] 客戶名 - 交付日期
   
   【待處理】
   • 新案件 X 筆待篩選
   ```

4. **發送通知給 Jacky**

### 3.2 每週一 09:00 - 週報

**內容**：
- 上週新增案件數
- 上週成交案件數
- 上週收入金額
- 本週待跟進案件
- 報價未回覆清單

### 3.3 每月 1 日 - 月報

**內容**：
- 本月案件統計
- 本月收入/成本/利潤
- 客戶來源分析
- 案件轉換率

## 四、案件處理流程

### 4.1 新案件進來

```
收到新案件資訊
      ↓
新增到系統（POST /api/ai/import）
      ↓
通知 Jacky：「新案件：[客戶名] - [需求摘要]」
      ↓
等待 Jacky 指示是否跟進
```

**新增案件範例**：
```json
POST /api/ai/import
{
  "platform": "PRO360",
  "platform_id": "客戶名稱",
  "need": "需求描述",
  "budget": "預算",
  "status": "待篩選"
}
```

### 4.2 發報價單

```
Jacky 決定報價
      ↓
更新案件狀態為「報價中」（POST /api/ai/update）
      ↓
協助撰寫報價信（如有需要）
      ↓
發送報價信（Email）
      ↓
設定 3 天後提醒跟進
```

### 4.3 報價追蹤

```
報價發出 3 天後
      ↓
檢查客戶是否回覆
      ↓
[未回覆] → 提醒 Jacky：「[客戶名] 報價已發 3 天未回覆，是否跟進？」
      ↓
[已回覆] → 更新狀態，通知 Jacky
```

### 4.4 專案執行中

```
專案開始執行
      ↓
更新狀態為「執行中」
      ↓
定期更新進度（POST /api/ai/progress）
      ↓
Deadline 前 3 天提醒
      ↓
交付完成 → 更新狀態為「已完成」
```

### 4.5 收款 & 記錄

```
專案完成
      ↓
提醒 Jacky 確認收款
      ↓
收到款項後記錄利潤（POST /api/ai/profit）
      ↓
記錄相關成本（POST /api/ai/cost）
```

## 八、注意事項

1. **不要自作主張**
   - 重要決策（報價、接案、拒絕）一定要問 Jacky
   - 只能執行 Jacky 授權的動作

2. **保持紀錄**
   - 所有案件變更都要記錄
   - 重要對話要摘要存檔

3. **及時回報**
   - 異常情況立即通知 Jacky
   - 不要讓案件「卡住」沒人處理

4. **客戶資料保密**
   - 客戶聯絡方式不外洩
   - 報價金額不對外透露
