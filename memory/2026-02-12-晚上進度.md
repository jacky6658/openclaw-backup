
## 🛠️ 技術架構

### 多渠道搜尋策略

| 職缺類型 | 主渠道 | 次渠道 | 預期結果 |
|----------|--------|--------|----------|
| **技術職**（AI 工程師） | GitHub (50%) | LinkedIn (30%) + 履歷池 (20%) | 30-50 人 |
| **非技術**（財會主管） | LinkedIn (70%) | 履歷池 (20%) + Google (10%) | 20-30 人 |
| **高階主管**（VP） | LinkedIn (60%) | 目標公司官網 (30%) + 履歷池 (10%) | 15-25 人 |

### 搜尋流程
```
1. 讀取 JD
2. AI 判斷職缺類型（技術 vs 非技術）
3. 選擇最佳渠道組合
4. 平行搜尋（web_search + GitHub API + 履歷池配對）
5. AI 統一配對 + 排序
6. 推測 Email（firstname.lastname@company.com）
7. 輸出 Top 20 + Telegram 通知
```
