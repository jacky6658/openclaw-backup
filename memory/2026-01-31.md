## 合作方式（重要約定）

### 1. 里程碑 + 階段交付
- Jacky 不希望一直問「寫好了沒」，也不希望我一直回狀態廢話。
- 我們約定：
  - 把工作拆成一個一個「里程碑 / 階段」（例如：學生讀書教練 MVP、老師課程列表、老師讀書計畫建議 UI、後端規格）。
  - 我實際完成某一段程式／文件後：
    - 改檔案
    - 跑 `npm run build` 確認不爆
    - 在 `dev-activity.log` 記錄一行（例如 `M1_DONE`, `M2_DONE`, `M3_DONE`）
    - 在聊天裡一次交付：「改了什麼檔案、功能差異、如何驗收」
  - 他只要有空時回來看「完成訊息」，照步驟驗收，不用一直打『開始』『下一步』。

### 2. 他回覆完成訊息 = 授權我做下一步
- 當我發出某個階段的「完成說明」後，只要他在那則訊息底下回：
  - 「OK」
  - 「看到了」
  - 或任何明確的正向回覆
- 我就視為：**可以繼續做下一個已排好的階段**，不用他再打「下一步」。

### 3. 不再承諾「背景一次做完很多件」
- 現實限制：我無法在他完全不在線的狀態下，可靠地連續完成 #2~#5 這種大包，然後只在最後一次報告。
- 因此，我不再承諾那種工作模式。
- 我能做到且會遵守的是：
  - 一個小里程碑一個交付，保證有實際改動、build 過、可驗收。
