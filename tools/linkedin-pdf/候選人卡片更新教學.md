# 候選人卡片更新教學
> Step1ne API v3.0 — 實戰筆記

**Base URL:** `https://backendstep1ne.zeabur.app`  
**actor 必填：** 每個 request body 都要加 `"actor": "Jacky-aibot"`

---

## ⚠️ 重要：兩個端點，功能不同

| 端點 | 方法 | 用途 |
|------|------|------|
| `/api/candidates/:id` | **PUT** | 更新 status / notes / aiMatchResult |
| `/api/candidates/:id` | **PATCH** | 更新其他所有欄位（skills, workHistory, targetJobId 等）|

**不能混用！** aiMatchResult 只能用 PUT，target_job_id 只能用 PATCH。

## 4. 指派職缺（PATCH）

```bash
curl -X PATCH "https://backendstep1ne.zeabur.app/api/candidates/1881" \
  -H "Content-Type: application/json" \
  -d '{
    "actor": "Jacky-aibot",
    "target_job_id": 8
  }'
```

> ⚠️ 注意：用 `target_job_id`（snake_case），不是 `targetJobId`

## 8. Python 批次更新範例

```python
import requests

API = "https://backendstep1ne.zeabur.app"

# 批次更新狀態
candidates = [
    {"id": 2080, "status": "AI推薦", "talent_level": "A"},
    {"id": 2082, "status": "備選人才", "talent_level": "B"},
]

for c in candidates:
    r = requests.put(f"{API}/api/candidates/{c['id']}", json={
        "actor": "Jacky-aibot",
        "status": c["status"],
    })
    print(f"[{c['id']}] {r.status_code}")
```

## 常見錯誤

| 問題 | 原因 | 解決 |
|------|------|------|
| `target_job_id` 沒存進去 | 用了 PUT 而非 PATCH | 改用 PATCH |
| `aiMatchResult` 沒存進去 | 用了 PATCH 而非 PUT | 改用 PUT |
| notes 被清空 | PUT 會覆蓋 notes | 先 GET 取舊值再帶入 |
| `{"error":"No fields to update"}` | 欄位名稱錯誤 | 確認用 snake_case 或正確 camelCase |
