## 📋 統一處理流程詳細步驟

### 步驟 1：收集所有候選人來源

**輸入**：職缺 ID（如「AI工程師」）

**來源清單**：
1. **履歷池**（被動）
   - 從 Google Sheets 讀取現有 240+ 份履歷
   - 根據技能關鍵字初步篩選
   
2. **GitHub**（主動）
   - API 搜尋：`location:Taiwan + 職位關鍵字`
   - 提取：用戶名、GitHub URL、repos、followers
   
3. **LinkedIn**（主動）
   - Brave Search：`職位 Taiwan site:linkedin.com/in`
   - agent-browser 訪問公開個人檔案
   - 提取：姓名、公司、技能

**輸出**：`all-candidates.json`（所有候選人清單）

---

### 步驟 2：去重處理

**功能**：跨平台合併 + 避免重複推薦

**去重邏輯**：
1. **生成候選人指紋**
   - 優先：email 的 MD5 hash
   - 次要：標準化姓名 + 標準化公司名
   - 備用：LinkedIn/GitHub URL hash

2. **合併重複候選人**
   - 同一指紋 → 合併資訊
   - 技能取聯集
   - 保留更完整的資料

3. **過濾已推薦**
   - 檢查 `data/recommended-history.json`
   - 同一職缺已推薦過的候選人不再推薦
   - 避免重複聯繫

**輸出**：`deduped.json`（去重後候選人）

---

### 步驟 3：AI 配對評分

**評分模型**（0-100 分）：

```
總分 = 技能匹配度 (40%) + 經驗年資 (30%) + 產業相關性 (20%) + 加分項 (10%) - 紅旗扣分
```

**詳細計算**：

**A. 技能匹配度（0-40 分）**
- 必備技能完全符合：40 分
- 必備技能部分符合：20-35 分（按比例）
- 加分技能符合：+5-10 分

**B. 經驗年資（0-30 分）**
| 情況 | 分數 |
|------|------|
| 剛好符合要求 | 30 |
| 高 1-2 年 | 28 |
| 高 3 年以上（overqualified）| 25 |
| 低 1 年 | 20 |
| 低 2 年 | 10 |
| 低 3 年以上 | 5 |

**C. 產業相關性（0-20 分）**
| 情況 | 分數 |
|------|------|
| 同產業同職位 | 20 |
| 同產業不同職位 | 15 |
| 相關產業同職位 | 10 |
| 無相關 | 0 |

**D. 加分項（0-10 分）**
- GitHub 活躍（followers >20）：+5
- 知名公司背景（A 級）：+3
- 知名公司背景（B 級）：+2
- 技術社群活躍：+2

**E. 紅旗扣分**
- 頻繁跳槽（1 年 3 家以上）：-15
- 技能完全不符（匹配率 <30%）：-20
- 地點不符且無遠端經驗：-10

**信心分級**：
- **P0（高度符合）**：≥80 分
- **P1（可能符合）**：60-79 分
- **P2（待確認）**：40-59 分
- **REJECT（不推薦）**：<40 分

**輸出**：`matched.json`（配對結果 + 分數 + 分級）

---

### 步驟 4：匯入履歷池 + Pipeline 追蹤

**匯入規則**：

## 🤖 主動學習系統

### 功能

**1. 記錄決策**
- Telegram 互動：`聯繫 王小明` / `略過 李小華`
- 自動記錄到 `data/decision-log.json`
- 包含：候選人、分數、特徵、決策時間

**2. 偏好分析（每週一執行）**
分析過去 7 天的決策，識別模式：
- GitHub 活躍偏好（你更常選 GitHub 活躍的人）
- 知名公司偏好（你更常選 A 級公司背景）
- 技能匹配要求（平均匹配度）
- 最低分數門檻（你最低聯繫幾分的候選人）

**3. 權重自動調整**
根據偏好分析建議調整評分權重：
- GitHub 活躍率 >70% → 提高 bonus 權重（0.10 → 0.15）
- 技能匹配率 >80% → 提高 skill_match 權重（0.40 → 0.45）
- 知名公司率 >60% → 增加公司加分（+3 → +5）

**4. 每週學習報告**
- 時間：每週一 09:00
- 發送：Topic 326（總覽看板）
- 內容：決策統計、偏好分析、權重調整建議
