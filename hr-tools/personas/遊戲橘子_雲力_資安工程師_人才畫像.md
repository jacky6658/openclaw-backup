## 🎯 人才畫像 (Candidate Persona)

### 人才類型
**「DevSecOps 守護者」** - 兼具開發與資安雙重視角的技術實踐者

### 代表心聲
> "我不只寫程式，我要寫出安全的程式。攻防兼備才是真正的工程師。"

### 人物描述
這類人才通常從 Web/App 開發起步，在實戰中逐漸發現資安的重要性，進而投入 DevSecOps 領域。他們不滿足於單純的功能開發，更在意如何在 CI/CD 流程中嵌入資安檢測，讓安全成為產品 DNA 的一部分。對他們而言，每一次部署都是一場攻防演練。

### 2️⃣ 人才特質 (Traits)
- 🔍 **好奇心強** - 喜歡探索漏洞、研究攻擊手法

### 3️⃣ 專業技能 (Skills)
**核心技能**：
- **程式開發**：Python / Node.js / Java / Go（至少一種）
- **Web 安全**：OWASP Top 10、SQL Injection、XSS、CSRF
- **DevOps 工具**：Docker / Kubernetes / Jenkins / GitLab CI
- **資安工具**：SAST / DAST / SCA（如 SonarQube、OWASP ZAP、Snyk）
- **雲端安全**：AWS / GCP / Azure 的 IAM、Security Groups、Secrets Management

**加分技能**：
- **滲透測試經驗**（Burp Suite、Metasploit）
- **Container Security**（Trivy、Clair、Falco）
- **API 安全**（OAuth、JWT、Rate Limiting）
- **Compliance 知識**（ISO 27001、PCI DSS）

### 4️⃣ 價值觀 (Values)
- 💡 **技術卓越** - 追求寫出「安全 + 高效」的程式碼
- 🌍 **社群貢獻** - 參與 OWASP、HITCON、資安社群活動
- 🎓 **知識分享** - 願意教育團隊資安意識（非自掃門前雪）
- ⚖️ **倫理駭客** - 有攻擊技能但用於防守，遵守道德規範

## 🔍 面試問題建議

### 技術面
1. **OWASP Top 10**：請說明 SQL Injection 的原理與防禦方法
2. **DevOps 整合**：如何在 CI/CD pipeline 中加入資安掃描？
3. **Container 安全**：Docker image 有哪些常見的資安風險？
4. **API 安全**：JWT 的安全性考量是什麼？如何防止 token 被盜用？

### 實戰面
5. **Code Review**：給一段有漏洞的程式碼，請找出問題並建議修正
6. **資安事件應對**：如果發現 production 有 SQL Injection 漏洞，你會怎麼處理？
7. **工具使用**：你用過哪些 SAST / DAST 工具？各自的優缺點是什麼？

### 文化面
8. **學習動機**：為什麼從開發轉資安？未來想成為什麼樣的專家？
9. **團隊合作**：如何說服開發團隊修復資安漏洞（他們可能覺得「沒那麼嚴重」）？
10. **證照規劃**：有考慮考取哪些資安證照嗎？
