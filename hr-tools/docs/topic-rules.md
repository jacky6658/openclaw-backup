## Topic 4: #1履歷進件
**用途**：新履歷上傳與自動匹配

**操作方式**：
1. **上傳履歷檔案**（PDF/Word/TXT）
2. **AI 自動處理**：
   - 提取履歷資訊
   - 與 7 個 JD 進行匹配
   - 計算匹配度分數
3. **自動通知**：
   - 🟢 高匹配（≥90%）：建議直接推薦
   - 🟡 中匹配（70-89%）：需顧問確認
   - ⚪ 低匹配（<70%）：放入履歷池

**執行時間**：
- 每小時自動檢查（09:00, 10:00, 11:00...）
- 手動上傳後可 @YuQi 立即處理

**注意事項**：
- ⚠️ 必須包含「👔 負責顧問」欄位
- 履歷會自動儲存到 Google Drive
- 匹配結果會推送到此 Topic

## Topic 304: #2履歷池
**用途**：履歷資料庫管理與搜尋

**功能**：
- 📂 儲存所有收到的履歷
- 🔍 搜尋候選人（關鍵字/技能/經驗）
- 📊 履歷統計報表
- 🔄 更新候選人狀態

**操作方式**：
- 搜尋履歷：`搜尋 [關鍵字]`
- 更新狀態：`更新狀態 [行數] [新狀態]`
- 查看報表：`履歷報表`

**履歷狀態**：
- 待聯繫 → 已聯繫 → 面試中 → 已錄取 → 已報到

**注意事項**：
- 低匹配度履歷自動進入此池
- 可隨時查詢過往候選人
- Google Sheets 同步更新

## Topic 364: #BD開發
**用途**：BD 客戶開發自動化

**自動執行**：
- ⏰ 每天 09:30 + 14:30 自動檢查
- 📧 發現「待寄信」狀態 → 自動發送 BD 信
- 📝 更新狀態為「已寄出 (日期)」

**手動操作**：
- 觸發自動開發：`@YuQi 開發客戶：[職位名稱]`
- 執行流程：
  1. 搜尋 104 招聘公司
  2. 爬取聯絡方式
  3. 整理到 BD客戶開發表
  4. 批量寄 BD 信
  5. 回報結果到此 Topic

**BD 信內容**：
- 自動署名「招募顧問 Jacky」
- 包含 Step1ne 公司簡介（PDF）
- AI 驅動履歷匹配系統介紹

**注意事項**：
- BD 表狀態：待寄信 → 已寄出 → 已回覆 → 合作中
- 已寄出的不會重複發送
- 追蹤回覆率與成交率

## Topic: 總覽看板
**用途**：Pipeline 視覺化與進度追蹤

**功能**：
- 📊 即時顯示招募 Pipeline
- 🎯 候選人狀態分布
- 📈 成交率與轉換率統計
- ⏰ 需跟進案件提醒

**查看方式**：
- Web 介面：http://localhost:3000（本地）
- 啟動指令：`~/clawd/hr-tools/start-dashboard.sh`
- 自動更新：每 30 秒刷新

**Pipeline 階段**：
1. 履歷收集
2. 初步篩選
3. 推薦給客戶
4. 安排面試
5. Offer 階段
6. 成功到職

**注意事項**：
- 數據來源：Google Sheets
- 需要 Zeabur 部署才能外網訪問
- 可設定提醒通知
