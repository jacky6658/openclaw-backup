## 📊 一、工作穩定性評分（Stability Score）

### 基本計算公式

```javascript
穩定度 = 基礎分 + 年資加分 - 離職次數扣分 + 最後離職間隔加分

基礎分：40 分
年資加分：總年資 × 2
離職次數扣分：離職次數 × 3
最後離職間隔加分：間隔月數 × 0.5
```

### 特殊案例處理

#### 案例 1：社會新鮮人（年資 < 1 年）
- 給予預設分數：**70 分（B級）**
- 理由：沒有負面記錄，給予信任基礎分

#### 案例 2：只有一份工作且仍在職（離職次數 = 0）
- 公式：`40 + (年資 × 3)`（年資加權提高）
- 上限：95 分
- 理由：從未離職是最穩定的表現

#### 案例 3：一般候選人（有離職記錄）
- 使用標準公式
- 範圍：20-100 分

## 🎯 二、綜合評級（Talent Grade）

### 評分維度（6個，總分100）

**權重分配**：

```javascript
const TALENT_GRADE_WEIGHTS = {
  skills: 25,        // 技能匹配度（業界最重視 - 能否快速上手）
  trajectory: 25,    // 職涯發展軌跡（成就與發展潛力）
  stability: 20,     // 工作穩定性（留任率）
  experience: 15,    // 工作年資（經驗基礎）
  education: 10,     // 學歷背景（門檻要求）
  bonus: 5          // 特殊加分（軟實力）
};
```

## 3️⃣ 技能匹配度（25%）

**評分邏輯**：

- **基礎分**：技能數量 (每個 1.5 分，上限 15 分)
- **資深加分**：技能中包含「架構」、「Lead」、「資深」、「專家」等字詞 → +5 分
- **認證加分**：包含「AWS」、「GCP」、「Azure」、「PMP」、「CISSP」、「證照」等 → +5 分
- **上限**：25 分

**範例**：
- `"Python, Java, React"` → 3個技能 → 4.5分
- `"Python, Java, React, AWS, Docker, Kubernetes"` → 6個技能 → 9分
- `"Senior Architect, Python, AWS Certified, Kubernetes, Docker, CI/CD"` → 6個 + 資深 + 認證 → 19分

---

## 4️⃣ 工作穩定性（20%）

直接使用「工作穩定性評分」：
```javascript
穩定度 0-100 分 → 轉換為 0-20 分
```

**範例**：
- 穩定度 70 分 → 14 分
- 穩定度 44 分 → 8.8 分
- 穩定度 95 分 → 19 分

## 5️⃣ 職涯發展軌跡（25%）

### 職位層級定義表

```
10級：CEO、CTO、CFO、COO、執行長、技術長、財務長、營運長
9級：總經理、VP、副總、Vice President、General Manager
8級：協理、總監、Director
7級：經理、Manager、部門主管、Department Head
6級：副理、組長、Team Lead、主管、Supervisor、資深專員、資深工程師
5級：工程師、Engineer、開發、Developer、專員、Specialist、Analyst、分析師
4級：初級工程師、Junior Engineer、Junior Developer
3級：助理、Assistant、助理專員
2級：實習生、Intern、工讀生
```

### 職涯軌跡分析

**分類**：
- **跨級晉升（3+ 級）**：+25 分（例：工程師 → 總監）
- **明顯晉升（2 級）**：+20 分
- **小幅晉升（1 級）**：+15 分
- **平級轉換**：+10 分
- **小幅降級（-1 級）**：+5 分
- **明顯降級（-2 級以上）**：+0 分

**加權獎勵**：
- 晉升次數 ≥ 3：額外 +5 分
- 無降級：額外 +3 分
- 上限：25 分

**職涯類型**：
- `fast_growing`：晉升 ≥ 2 次
- `growing`：晉升 > 平級轉換
- `mixed`：有降級
- `stable`：無晉升但穩定

## 6️⃣ 特殊加分（5%）

**軟實力加分項**：

| 項目 | 加分 | 關鍵字 |
|------|------|--------|
| 語言能力 | +2 分 | 英文、雙語、bilingual、TOEIC、IELTS、TOEFL、CELPIP |
| 軟實力 | +2 分 | 溝通、communication、領導、leadership、團隊合作、teamwork、問題解決、適應力 |
| 特殊成就 | +1 分 | 獲獎、award、專利、patent、出版、publication、演講、speaker、conference |

**上限**：5 分

## 📈 綜合評級計算結果

### 評級對照表

```
90-100 分 → S 級   （頂尖人才）
80-89  分 → A+ 級  （優秀人才）
70-79  分 → A 級   （合格人才）
60-69  分 → B 級   （潛力人才）
< 60   分 → C 級   （需培訓）
```

## 📋 實際案例

### 案例 1：資深技術人才（王大明）

**候選人資料**：
```
姓名：王大明
學歷：碩士（計算機科學）
年資：8 年
離職次數：1 次（穩定度 78 分）
技能：Python, TensorFlow, PyTorch, AWS Certified, Docker, Kubernetes, CI/CD
工作經歷：
  1. 資深 AI 工程師（6級）2021-2026
  2. AI 工程師（5級）2018-2021
  3. 初級工程師（4級）2016-2018
備註：雙語能力、技術演講者
```

**評分計算**：
```
學歷（碩士）：         9分      ×10% = 0.9分
年資（8年）：          12.5分   ×15% = 1.875分
技能（7個+認證）：      19分      ×25% = 4.75分
穩定度（78分）：       15.6分   ×20% = 3.12分
職涯軌跡（2次晉升）：   25分      ×25% = 6.25分
特殊加分（雙語+演講）： 3分      ×5% = 0.15分
───────────────────────────────────────
總分：17.075 分... 等等，我算錯了

正確計算：
9 + 12.5 + 19 + 15.6 + 25 + 3 = 84.1 分 → A+ 級
```

✅ **評價**：優秀人才

```
  3. 經理（7級）2010-2015
```


✅ **評價**：完美！高階人才應得高分。
