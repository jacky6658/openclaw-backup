# 自動化爬人選前端系統設計方案

**需求**：有用戶前端介面的自動化爬人選腳本  
**日期**：2026-03-04

---

## 🎯 核心功能需求

### 用戶介面需要什麼？

1. **輸入區**
   - 職位名稱（如：Java Developer）
   - 公司名稱（可選）
   - 技能要求（標籤選擇 + 自由輸入）
   - 年資要求（下拉選單）
   - 地點（台北/新竹/台中等）
   - 搜尋數量（10/20/50/100）

2. **平台選擇**
   - [ ] LinkedIn
   - [ ] GitHub
   - [ ] 104 人力銀行
   - [ ] CakeResume
   - [ ] PTT Soft_Job

3. **進度顯示**
   - 即時進度條
   - 當前步驟（搜尋中/去重中/評分中）
   - 已找到人數
   - 錯誤提示

4. **結果展示**
   - 候選人卡片（頭像、姓名、技能、分數）
   - 篩選排序（依分數、來源、技能）
   - 匯出功能（CSV / 匯入系統）

## 📋 方案比較

### 方案 A：超簡單版（推薦快速啟動）⭐

**技術棧**：
- 前端：純 HTML + Tailwind CSS + Alpine.js（輕量）
- 後端：Python Flask（極簡 API）
- 爬蟲：直接呼叫現有 `unified-scraper-v4.py`

**優點**：
- ✅ 1-2 小時完成
- ✅ 無需安裝 Node.js
- ✅ 單一 HTML 檔案即可運行
- ✅ 輕量、快速

**缺點**：
- ⚠️ 功能較陽春
- ⚠️ UI 美化程度有限

**適合**：
- 內部使用
- 快速驗證想法
- MVP 測試

---

### 方案 B：現代化版（平衡效能與開發速度）⭐⭐

**技術棧**：
- 前端：React + Vite + Tailwind CSS
- 後端：FastAPI (Python) + WebSocket
- 爬蟲：整合 `one-bot-pipeline.py`

**優點**：
- ✅ 現代化 UI（拖拉、動畫）
- ✅ 即時進度更新（WebSocket）
- ✅ 響應式設計（手機可用）
- ✅ 可擴展性強

**缺點**：
- ⚠️ 需要 3-4 小時開發
- ⚠️ 需要 npm + Python 環境

**適合**：
- 正式產品
- 多人使用
- 長期維護

### API 設計

```python
# Flask API 端點

POST /api/search
{
  "job_title": "Java Developer",
  "company": "台積電",
  "skills": ["Java", "Spring Boot"],
  "experience": "3-5年",
  "location": "台北",
  "limit": 20,
  "platforms": ["linkedin", "github"]
}

→ 回傳: {
  "task_id": "abc123",
  "status": "processing"
}

GET /api/search/status/<task_id>
→ 回傳進度

GET /api/search/result/<task_id>
→ 回傳結果

POST /api/import
→ 匯入候選人到 Step1ne 系統
```
