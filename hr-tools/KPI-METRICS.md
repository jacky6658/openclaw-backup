## 🎯 职缺优先级指标

### 综合评分公式（满分 100 分）
```
总分 = 成交速度(40%) + 佣金收益(30%) + 搜索成本(15%) + 成交机率(10%) + 客户稳定(5%)
```

### 1. 成交速度评分（40 分）
**目标**：越快成交越优先

| 评估项目 | 满分 | 评分标准 |
|---------|------|----------|
| 年资要求 | 10分 | 0-1年=10分 / 1-2年=8分 / 2-3年=6分 / 3年+=4分 |
| 技术门槛 | 10分 | 通用技术=10分 / 常见技术=8分 / 专精技术=5分 / 稀缺技术=3分 |
| 市场供给量 | 10分 | >200人/月=10分 / 100-200=8分 / 50-100=6分 / <50=4分 |
| 履历库匹配度 | 5分 | 已有10+人=5分 / 5-10人=4分 / 1-5人=2分 / 0人=0分 |
| 客户配合度 | 5分 | 快速决策=5分 / 正常速度=3分 / 缓慢=1分 |

**计算范例**：
```
Java Developer（一通数位）
- 年资要求（2年）= 6分
- 技术门槛（常见）= 8分
- 市场供给量（>200）= 10分
- 履历库匹配（26人）= 5分
- 客户配合度（未知）= 3分
→ 小计：32/40 分
```

---

### 2. 佣金收益评分（30 分）
**目标**：优先高佣金职缺

**计算公式**：
```
佣金收益分 = (预估总佣金 / 10,000) 分（上限 30 分）

预估总佣金 = 平均薪资 × 佣金率 × 职缺人数
```

| 总佣金区间 | 得分 | 说明 |
|-----------|------|------|
| ≥30万 | 30分 | 高价值（协理级以上） |
| 20-30万 | 25分 | 中高价值（经理级多人） |
| 10-20万 | 20分 | 中等价值（工程师多人） |
| 5-10万 | 15分 | 基础价值（单人中阶） |
| <5万 | 10分 | 低价值（初阶单人） |

**计算范例**：
```
Java Developer（一通数位）
- 平均薪资：7.5万
- 佣金率：15%
- 职缺人数：2.5人（2-3人平均）
→ 预估总佣金：7.5万 × 15% × 2.5 = 2.8125万
→ 得分：2.8125 × (30/3) ≈ 27分
```

---

### 3. 搜索成本评分（15 分）
**目标**：优先低成本管道

| 评估项目 | 满分 | 评分标准 |
|---------|------|----------|
| 管道适配度 | 7分 | 免费管道=7分 / 低成本API=5分 / 付费管道=3分 |
| API配额消耗 | 4分 | 无需API=4分 / 少量API=3分 / 大量API=1分 |
| 人工筛选成本 | 4分 | AI自动=4分 / 半自动=2分 / 全手动=0分 |

**成本参考**：
```
- GitHub 免费搜索：NT$ 0
- LinkedIn 手动搜索：NT$ 0（时间成本）
- Apollo.io Free：NT$ 0（10k credits/月）
- ContactOut $49：NT$ 1,570/月
- LinkedIn Recruiter：NT$ 35,000/月
```

**计算范例**：
```
Java Developer（一通数位）
- 管道：GitHub（免费）+ LinkedIn（已有候选人）= 7分
- API：无需额外 API = 4分
- 筛选：AI 自动评分 = 2分（需人工确认）
→ 小计：13/15 分
```

---

### 4. 成交机率评分（10 分）
**目标**：优先高成交率职缺

| 评估项目 | 满分 | 评分标准 |
|---------|------|----------|
| 履历库合格候选人 | 4分 | ≥10人=4分 / 5-10人=3分 / 1-5人=2分 / 0人=0分 |
| 市场竞争度 | 3分 | 低竞争=3分 / 中竞争=2分 / 高竞争=1分 |
| 职缺吸引力 | 3分 | 高吸引=3分 / 中吸引=2分 / 低吸引=1分 |

**职缺吸引力参考**：
```
高吸引力：知名品牌 + 高薪 + 好福利 + 远端弹性
中吸引力：一般公司 + 合理薪资 + 标准福利
低吸引力：小公司 + 低薪 + 无特色福利
```

---

### 5. 客户稳定度评分（5 分）
**目标**：优先长期合作客户

| 评估项目 | 满分 | 评分标准 |
|---------|------|----------|
| 多职缺客户 | 3分 | ≥5职缺=3分 / 3-5职缺=2分 / 1-2职缺=1分 |
| 长期合作 | 2分 | 老客户=2分 / 新客户=1分 |

## 👔 顾问绩效指标

### 核心 KPI（每周/每月追踪）

| 指标 | 计算公式 | 目标值（周） | 目标值（月） |
|------|----------|-------------|-------------|
| **联系候选人数** | 新增联系人数 | 20-30 人 | 80-120 人 |
| **有效回复率** | 回复数 / 联系数 | ≥30% | ≥30% |
| **送履历数** | 实际送出履历数 | 5-8 份 | 20-30 份 |
| **面试安排数** | 客户邀约面试数 | 2-4 次 | 8-15 次 |
| **成交数** | 候选人录取数 | 0.5-1 人 | 2-4 人 |
| **成交率** | 成交数 / 送履历数 | ≥10% | ≥10% |
| **总佣金** | 成交佣金总额 | - | NT$ 50k-100k |

### 进阶指标（质量追踪）

| 指标 | 计算公式 | 目标值 |
|------|----------|--------|
| **平均成交天数** | 从首次联系到录取的天数 | ≤21 天 |
| **候选人满意度** | 候选人反馈评分（1-5） | ≥4.0 |
| **客户满意度** | 客户评分（1-5） | ≥4.5 |
| **履历转化率** | 面试数 / 送履历数 | ≥40% |
| **面试转化率** | 成交数 / 面试数 | ≥25% |

### 效率指标

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| **搜索效率** | 合格候选人 / 搜索总数 | AI 评分 A/B 级占比 |
| **资料完整度** | 有联络资讯候选人 / 总候选人 | 目标 ≥90% |
| **重复联系率** | 重复候选人数 / 新增数 | 越低越好（<10%） |

## 🎓 候选人质量指标

### AI 评分系统（100 分制）

| 等级 | 分数区间 | 说明 | 建议动作 |
|------|---------|------|----------|
| **S 级** | 90-100 | 完美匹配 | 立即联系、优先推荐 |
| **A+ 级** | 85-89 | 优秀候选人 | 24h 内联系 |
| **A 级** | 70-84 | 推荐级 | 48h 内联系 |
| **B 级** | 55-69 | 备选人才 | 储备、视情况联系 |
| **C 级** | 40-54 | 勉强合格 | 低优先级 |
| **D 级** | <40 | 不合格 | 不推荐 |

### 质量维度（AI 评分细项）

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **技能匹配度** | 35% | JD 技能 vs 候选人技能重叠度 |
| **年资匹配度** | 20% | 总年资 & 相关经验年资 |
| **稳定性评分** | 15% | 平均任职时长、转职频率、gap 分析 |
| **学历背景** | 10% | 学历要求匹配度 |
| **GitHub/作品集** | 10% | 技术影响力（stars/活跃度） |
| **LinkedIn 完整度** | 10% | 资料完整度、推荐人、证照 |

### 稳定性评分公式
```python
稳定性分数 = (
    平均任职月数 / 24 * 0.4 +          # 40%：平均任职时长
    (1 / 转职次数) * 0.3 +              # 30%：转职频率（越少越好）
    (1 - 最近gap月数/12) * 0.3          # 30%：gap 分析（越短越好）
) * 100

# 范例
平均任职 18 个月、转职 3 次、最近 gap 2 个月
= (18/24 * 0.4) + (1/3 * 0.3) + (1 - 2/12 * 0.3) * 100
= (0.75 * 0.4) + (0.33 * 0.3) + (0.83 * 0.3) * 100
= (0.3 + 0.1 + 0.25) * 100
= 65 分（中等稳定）
```

## 💰 财务指标

### ROI 计算公式

#### 1. 职缺 ROI（单一职缺）
```
职缺 ROI = (预估佣金 - 搜索成本) / 预估成交时间（天）

范例：Java Developer
预估佣金：NT$ 31,500（7.5万 × 15% × 2.8人）
搜索成本：NT$ 500（API + 工具）
成交时间：7 天

ROI = (31,500 - 500) / 7 = NT$ 4,428/天
```

#### 2. 顾问 ROI（个人绩效）
```
顾问月 ROI = 总佣金 / (固定成本 + 变动成本)

固定成本：薪资 + 福利 + 分摊成本
变动成本：工具订阅 + API 费用 + 其他支出

范例：
总佣金：NT$ 80,000
固定成本：NT$ 40,000
变动成本：NT$ 5,000
ROI = 80,000 / 45,000 = 1.78（每投入 1 元赚 1.78 元）
```

#### 3. 系统 ROI（整体效率）
```
系统月 ROI = 总佣金 / 总营运成本

总营运成本 = 人力成本 + 工具成本 + 基础设施 + 其他

目标 ROI：≥2.0（每投入 1 元赚 2 元以上）
```

### 成本追踪

| 成本项目 | 月成本（NT$） | 占比 | 说明 |
|---------|-------------|------|------|
| **人力成本** | 变动 | 60-70% | 顾问薪资 + 福利 |
| **工具订阅** | 1,570-35,000 | 5-15% | ContactOut/LinkedIn/Apollo |
| **API 费用** | 500-2,000 | 1-5% | GitHub/Google Sheets/其他 |
| **基础设施** | 1,000-3,000 | 2-5% | Zeabur/域名/储存 |
| **其他支出** | 1,000-5,000 | 2-10% | 行销/杂支 |

## 😊 客户满意度指标

### NPS（净推荐值）
```
NPS = % 推荐者（9-10分） - % 贬损者（0-6分）

目标 NPS：≥50（优秀）
```

### 客户满意度维度

| 维度 | 权重 | 衡量方式 |
|------|------|----------|
| **候选人质量** | 40% | 面试邀约率、录取率 |
| **回应速度** | 25% | 首次回应时间、问题解决时间 |
| **专业度** | 20% | 沟通品质、职缺理解度 |
| **售后服务** | 15% | 到职追踪、问题处理 |

### 客户留存指标

| 指标 | 计算公式 | 目标值 |
|------|----------|--------|
| **客户留存率** | 持续合作客户 / 总客户 | ≥70% |
| **复购率** | 重复下单客户 / 总客户 | ≥50% |
| **平均合作职缺数** | 总职缺 / 客户数 | ≥3 个 |

## 🎯 指标使用建议

### 职缺优先级决策
```
1. 计算所有「招募中」职缺的综合评分
2. 排序 TOP 10（≥70 分优先执行）
3. 每周检视一次，动态调整
```

### 顾问绩效评估
```
1. 每周一检视上周 KPI（联系/送履历/成交）
2. 每月底计算月度 ROI
3. 季度评估整体表现（奖金/晋升参考）
```

### 系统优化决策
```
1. 每周检视自动化效率（<80% 需优化）
2. 每月检视成本结构（工具订阅 ROI）
3. 季度检视整体系统 ROI（<2.0 需改善）
```
