## 1. 候選人狀態總覽

| 狀態值 | 說明 | Kanban 欄位 |
|--------|------|-------------|
| `未開始` | 新匯入，尚未處理 | 未開始 |
| `AI推薦` | AI 自動推薦 | AI推薦 |
| `聯繫階段` | 已聯繫候選人 | 聯繫階段 |
| `面試階段` | 安排或進行面試 | 面試階段 |
| `Offer` | 已發 Offer | Offer |
| `on board` | 已報到上班 | on board |
| `婉拒` | 候選人或公司婉拒 | 婉拒 |
| `備選人才` | 暫時保留觀望 | 備選人才 |

**移動規則**：
- 卡片可在 8 個狀態之間自由移動（無固定順序限制）
- 每次移動自動記錄到 `progress_tracking` + `system_logs`
- 移到「婉拒」時，務必附帶 `notes` 記錄原因

## 2. 情境一：拿到履歷 / 分析履歷

### 步驟 1｜查重（先查再動）

搜尋系統，確認候選人是否已存在：

```bash
# 用姓名搜尋
curl "https://backendstep1ne.zeabur.app/api/candidates/search?q=姓名"

# 用 Email 搜尋
curl "https://backendstep1ne.zeabur.app/api/candidates?email=xxx@gmail.com"

# 用 LinkedIn URL 搜尋
curl "https://backendstep1ne.zeabur.app/api/candidates?linkedinUrl=*keyword*"
```

| 結果 | 動作 |
|------|------|
| **已存在** | 用 `PATCH /api/candidates/:id` 更新資料 |
| **不存在** | 用 `POST /api/candidates` 新增 |

> ⚠️ **禁止直接新增而不先查重** — 會產生重複資料

---

### 步驟 2｜記錄指派

確認並記錄以下兩件事：

- **負責顧問**：Jacky 傳的 → `recruiter: "Jacky"`；Phoebe 傳的 → `recruiter: "Phoebe"`
- **目標職缺**：哪個 Job ID → `target_job_id: <JD編號>`

---

### 步驟 3｜解析履歷完整內容

從履歷中提取以下資料：

| 項目 | API 欄位 | 格式 |
|------|----------|------|
| 工作經歷（公司/職位/起迄） | `work_history` | JSONB 陣列（見第5節） |
| 教育背景 | `education_details` | JSONB 陣列（見第5節） |
| 技能 | `skills` | 逗號或頓號分隔字串 |
| 語言能力 | `skills` 或 `notes` | 附加說明 |
| 證照 | `skills` 或 `notes` | 附加說明 |
| 換工作次數 | `job_changes` | 數字字串 |
| 平均任職月數 | `avg_tenure_months` | 數字字串 |
| 近期空窗月數 | `recent_gap_months` | 數字字串 |

---

### 步驟 4｜5 維度 AI 評分

對候選人進行以下五個維度的評估，寫入 `ai_match_result`：

| 維度 | 說明 |
|------|------|
| ① 技術匹配 | 技能/工具/年資是否符合職缺需求 |
| ② 產業匹配 | 過去產業經歷與目標產業的重疊度 |
| ③ 公司背景 | 規模/文化/知名度是否對齊 |
| ④ 薪資對齊 | 期望薪資是否在客戶預算內 |
| ⑤ 求職動機 | 是否主動求職、離職原因是否合理 |

**評分結果對應**：

| recommendation | 分數區間 | 說明 |
|----------------|----------|------|
| `強力推薦` | 85-100 | 高度符合 |
| `推薦` | 70-84 | 符合 |
| `觀望` | 50-69 | 部分符合 |
| `不推薦` | 0-49 | 不符合 |

---

### 步驟 5｜匯入系統

```bash
# 新增候選人（附完整資料）
curl -X POST https://backendstep1ne.zeabur.app/api/candidates \
  -H "Content-Type: application/json" \
  -d '{
    "name": "王小明",
    "email": "wang@example.com",
    "phone": "0912345678",
    "location": "台北",
    "current_position": "Senior DevOps Engineer",
    "years_experience": "6",
    "skills": "Docker、Kubernetes、AWS、Jenkins",
    "source": "LinkedIn",
    "status": "AI推薦",
    "recruiter": "Jacky",
    "target_job_id": 53,
    "job_changes": "3",
    "avg_tenure_months": "24",
    "work_history": [...],
    "education_details": [...],
    "notes": "2026-03-05 收到，來源：Jacky，目標職缺：一通數位DevOps #53",
    "actor": "AIbot-Jacky"
  }'
```

**備註欄格式**：
```
[日期] 收到，來源：[Jacky/Phoebe]，目標職缺：[職缺名稱 #ID]
```

---

### 步驟 6｜AI 匹配結語（寫入 ai_match_result）

生成以下六個部分並寫入系統：

| 項目 | 說明 |
|------|------|
| **a. 系統內職缺匹配推薦** | 目前 JD 清單中最適合的 1-3 個職缺 |
| **b. 系統外職缺建議** | 依候選人背景，建議可開發的潛在客戶方向 |
| **c. 建議面談問題** | 針對此候選人的專屬探問題目（3-5 題） |
| **d. 人選分析報告** | 優勢、劣勢、穩定性評估 |
| **e. AI 完整結論** | |
| └ e1. 推薦原因 | 為什麼適合這份工作 |
| └ e2. 適合哪些職缺 | 含 JD ID 與職位名稱 |
| └ e3. 該問什麼問題 | 需要進一步確認的疑點 |
| └ e4. 優勢與劣勢 | 具體列點 |
| └ e5. 如何聯繫 | 套用職缺介紹庫模板，產出聯繫草稿 |

```bash
# 更新 AI 評估結果
curl -X PATCH https://backendstep1ne.zeabur.app/api/candidates/123 \
  -H "Content-Type: application/json" \
  -d '{
    "ai_match_result": {
      "score": 88,
      "recommendation": "強力推薦",
      "job_id": 53,
      "job_title": "一通數位 DevOps Engineer",
      "matched_skills": ["Docker", "Kubernetes", "AliCloud"],
      "missing_skills": ["Kafka"],
      "strengths": ["6年DevOps經驗", "AliCloud 實戰背景"],
      "probing_questions": ["有過 on-call 輪班經驗嗎？", "CI/CD 實際用哪套工具？"],
      "conclusion": "技術棧高度吻合，建議優先聯繫",
      "evaluated_at": "2026-03-05T10:00:00Z",
      "evaluated_by": "AIbot-Jacky"
    },
    "actor": "AIbot-Jacky"
  }'
```

## 3. 情境二：Jacky 回報聯繫結果 SOP

Jacky 回報任何聯繫/面談進展後，依以下分支執行對應 API 操作。

---

### 分支 1｜聯繫人選

**觸發**：Jacky 說「已聯繫 XXX」或「傳訊息給 XXX 了」

```bash
curl -X PUT https://backendstep1ne.zeabur.app/api/candidates/:id/pipeline-status \
  -H "Content-Type: application/json" \
  -d '{
    "status": "聯繫階段",
    "by": "Jacky",
    "notes": "2026-03-05 聯繫結果：[Jacky 回報的內容]"
  }'
```

| 欄位 | 值 |
|------|-----|
| Status | `聯繫階段` |
| 備註格式 | `[日期] 聯繫結果：[回報內容]` |

---

### 分支 2｜面談階段-1（安排面談）

**觸發**：Jacky 說「安排面談了」或「約好時間了」

```bash
curl -X PUT https://backendstep1ne.zeabur.app/api/candidates/:id/pipeline-status \
  -H "Content-Type: application/json" \
  -d '{
    "status": "面試階段",
    "by": "Jacky",
    "notes": "2026-03-05 面談安排：[實體面談 or 客戶公司面談]，時間：[日期]"
  }'
```

| 欄位 | 值 |
|------|-----|
| Status | `面試階段` |
| 備註格式 | `[日期] 面談安排：[實體/客戶公司面談]，時間：[日期]` |

---

### 分支 3｜面談階段-2（面談結果）

**觸發**：Jacky 說「面談完了」或「面試結果出來了，等待回音」

```bash
curl -X PATCH https://backendstep1ne.zeabur.app/api/candidates/:id \
  -H "Content-Type: application/json" \
  -d '{
    "notes": "2026-03-05 面談結果：[Jacky 回報內容]",
    "actor": "Jacky"
  }'
```

| 欄位 | 值 |
|------|-----|
| Status | **維持 `面試階段`**（不動，等待結果） |
| 備註格式 | `[日期] 面談結果：[回報內容]` |

> ⚠️ **此分支不移動狀態**，只更新備註

---

### 分支 4｜人選拿到 Offer

**觸發**：Jacky 說「錄取了」或「Offer 出去了」

```bash
curl -X PUT https://backendstep1ne.zeabur.app/api/candidates/:id/pipeline-status \
  -H "Content-Type: application/json" \
  -d '{
    "status": "Offer",
    "by": "Jacky",
    "notes": "2026-03-05 錄取：[職缺名稱]，報到日期：[日期]"
  }'
```

| 欄位 | 值 |
|------|-----|
| Status | `Offer` |
| 備註格式 | `[日期] 錄取：[職缺名稱]，報到日期：[日期]` |

---

### 分支 5｜人選沒錄取（客戶婉拒）

**觸發**：Jacky 說「客戶沒要」或「被婉拒了」

```bash
curl -X PUT https://backendstep1ne.zeabur.app/api/candidates/:id/pipeline-status \
  -H "Content-Type: application/json" \
  -d '{
    "status": "婉拒",
    "by": "Jacky",
    "notes": "2026-03-05 客戶婉拒，原因：[原因]"
  }'
```

| 欄位 | 值 |
|------|-----|
| Status | `婉拒` |
| 備註格式 | `[日期] 客戶婉拒，原因：[原因]` |

---

### 分支 6｜人選無面談意願

**觸發**：Jacky 說「人選沒興趣」或「不想動了」

```bash
curl -X PATCH https://backendstep1ne.zeabur.app/api/candidates/:id \
  -H "Content-Type: application/json" \
  -d '{
    "notes": "2026-03-05 人選無意願，原因：[原因]",
    "actor": "Jacky"
  }'
```

| 欄位 | 值 |
|------|-----|
| Status | **維持 `AI推薦`**（不移動） |
| 備註格式 | `[日期] 人選無意願，原因：[原因]` |

> ⚠️ **此分支不移動狀態** — 保留在 AI推薦，未來可能重新啟動

## 4. API 速查表

| 操作 | 方法 | 路徑 |
|------|------|------|
| 搜尋候選人 | `GET` | `/api/candidates/search?q=姓名` |
| 新增候選人 | `POST` | `/api/candidates` |
| 批量匯入 | `POST` | `/api/candidates/bulk`（最多 100 筆） |
| 更新候選人資料 | `PATCH` | `/api/candidates/:id` |
| 移動卡片狀態 | `PUT` | `/api/candidates/:id/pipeline-status` |
| 批量移動狀態 | `PATCH` | `/api/candidates/batch-status`（最多 200 筆） |
| 刪除候選人 | `DELETE` | `/api/candidates/:id` |

**欄位名稱彈性支援（camelCase / snake_case 皆可）**：

| 可用名稱 | DB 欄位 |
|----------|---------|
| `notes` 或 `remarks` | notes |
| `linkedin_url` 或 `linkedinUrl` | linkedin_url |
| `position` 或 `current_position` | current_position |
| `years` 或 `years_experience` | years_experience |
| `by` 或 `actor` | actor |
| `target_job_id` 或 `targetJobId` | target_job_id |

## 5. JSONB 欄位結構

### work_history — 工作經歷

```json
[
  {
    "company": "台積電",
    "title": "Senior Backend Engineer",
    "start": "2020-01",
    "end": "2022-06",
    "duration_months": 30,
    "location": "Hsinchu",
    "description": "主導微服務架構遷移"
  }
]
```

### education_details — 教育經歷

```json
[
  {
    "school": "國立台灣大學",
    "degree": "Bachelor",
    "major": "Computer Science",
    "start": "2012",
    "end": "2016"
  }
]
```

### ai_match_result — AI 配對評估

```json
{
  "score": 88,
  "recommendation": "強力推薦",
  "job_id": 53,
  "job_title": "一通數位 DevOps Engineer",
  "matched_skills": ["Docker", "Kubernetes", "AliCloud"],
  "missing_skills": ["Kafka"],
  "strengths": ["6年DevOps經驗", "AliCloud 實戰背景"],
  "probing_questions": ["有過 on-call 輪班經驗嗎？"],
  "conclusion": "技術棧高度吻合，建議優先聯繫",
  "evaluated_at": "2026-03-05T10:00:00Z",
  "evaluated_by": "AIbot-Jacky"
}
```

### progress_tracking — 進度追蹤（自動生成，勿手動傳入）

```json
[
  { "date": "2026-03-01", "event": "未開始", "by": "system" },
  { "date": "2026-03-05", "event": "聯繫階段", "by": "Jacky", "note": "已電話聯繫" },
  { "date": "2026-03-07", "event": "面試階段", "by": "Jacky" }
]
```

## 6. 完整 curl 範例

### 新增候選人（含 AI 評估）

```bash
curl -X POST https://backendstep1ne.zeabur.app/api/candidates \
  -H "Content-Type: application/json" \
  -d '{
    "name": "David Lee",
    "email": "david@example.com",
    "current_position": "Senior DevOps Engineer",
    "years_experience": "8",
    "skills": "Docker、Kubernetes、AWS、Terraform、Jenkins",
    "source": "LinkedIn",
    "recruiter": "Jacky",
    "target_job_id": 53,
    "ai_match_result": {
      "score": 92,
      "recommendation": "強力推薦",
      "job_id": 53,
      "job_title": "一通數位 DevOps",
      "matched_skills": ["Docker", "Kubernetes", "AWS"],
      "missing_skills": ["AliCloud"],
      "strengths": ["8年DevOps經驗", "強雲端基礎設施背景"],
      "probing_questions": ["有 AliCloud 使用經驗嗎？"],
      "conclusion": "頂級候選人，建議優先安排面談"
    },
    "work_history": [
      {
        "company": "Trend Micro",
        "title": "Senior DevOps Engineer",
        "start": "2022-03",
        "end": "present",
        "duration_months": 36
      }
    ],
    "notes": "2026-03-05 收到，來源：Jacky，目標職缺：一通數位DevOps #53",
    "actor": "AIbot-Jacky"
  }'
```

### 移動卡片狀態

```bash
curl -X PUT https://backendstep1ne.zeabur.app/api/candidates/123/pipeline-status \
  -H "Content-Type: application/json" \
  -d '{
    "status": "面試階段",
    "by": "Jacky",
    "notes": "2026-03-05 面談安排：客戶公司面談，時間：2026-03-08"
  }'
```

### 只更新備註（不動狀態）

```bash
curl -X PATCH https://backendstep1ne.zeabur.app/api/candidates/123 \
  -H "Content-Type: application/json" \
  -d '{
    "notes": "2026-03-05 面談結果：表現不錯，客戶評估中",
    "actor": "Jacky"
  }'
```

### 批量移動狀態

```bash
curl -X PATCH https://backendstep1ne.zeabur.app/api/candidates/batch-status \
  -H "Content-Type: application/json" \
  -d '{
    "ids": [123, 124, 125],
    "status": "聯繫階段",
    "actor": "Jacky-aibot",
    "note": "已完成初步篩選，統一更新"
  }'
```

---

> 📌 **注意事項**：
> - `status` 值必須完全匹配上方 8 個合法值之一
> - `progress_tracking` 為自動追蹤，**不需手動傳入**
> - 批量操作限制：匯入 100 筆/次、狀態更新 200 筆/次
> - 候選人姓名為唯一識別（大小寫不敏感），重複匯入自動合併
