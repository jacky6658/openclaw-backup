# EduMind Study Plans & Tasks - Backend Spec (Draft)

## Overview

This document defines the **database schema** and **REST API** for managing
student study plans and tasks in EduMind.

Goals:
- Support **per-student study plans** (generated by coach / AI / teachers)
- Track **per-day study tasks** with status + duration
- Work well in a **multi-tenant** environment (school / workspace)
- Be simple enough to implement in a first backend

Assumptions:
- DB: PostgreSQL
- Multi-tenant key: `tenant_id` on all tenant-scoped tables
- Auth / user tables already exist (e.g. `users`, `tenants`, `courses`)

---

## Core Tables

### 1. study_plans

每個學生在某個時間範圍內的一份「讀書計畫」主檔。

```sql
CREATE TABLE study_plans (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES tenants(id),
  student_id      UUID NOT NULL REFERENCES users(id),
  course_id       UUID NULL REFERENCES courses(id),

  -- 計畫範圍
  title           TEXT NOT NULL,            -- 例如「學測前四週衝刺計畫」
  target_exam_date DATE NULL,              -- 目標考試日期（可選）
  start_date      DATE NOT NULL,
  end_date        DATE NOT NULL,

  -- 由誰建立
  created_by_role TEXT NOT NULL,           -- 'student' | 'teacher' | 'ai' | 'admin'
  created_by_id   UUID NULL,               -- optional reference to users(id)

  -- 統計 / 設定快照
  planned_days    INTEGER NOT NULL,        -- 計畫涵蓋的天數
  planned_total_hours NUMERIC(5,2) NOT NULL,

  -- 狀態
  status          TEXT NOT NULL DEFAULT 'ACTIVE',
                   -- 'ACTIVE' | 'ARCHIVED' | 'CANCELLED' | 'COMPLETED'

  -- metadata
  config_json     JSONB NULL,              -- e.g. 原始設定 (每週幾天、每天幾小時、優先科目)
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_study_plans_student ON study_plans(tenant_id, student_id, status);
CREATE INDEX idx_study_plans_course  ON study_plans(tenant_id, course_id);
```

### 2. study_tasks

計畫下的單日 / 單時段任務。

```sql
CREATE TABLE study_tasks (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES tenants(id),
  plan_id         UUID NOT NULL REFERENCES study_plans(id) ON DELETE CASCADE,

  task_date       DATE NOT NULL,           -- 任務日期
  subject_code    TEXT NOT NULL,           -- 'python' | 'math' | ...
  title           TEXT NOT NULL,           -- e.g. "複習第 3 章 + 完成習題 3-1 ~ 3-5"
  description     TEXT NULL,

  duration_hours  NUMERIC(4,2) NOT NULL,   -- 預計時數
  order_in_day    INTEGER NOT NULL DEFAULT 1,

  -- 狀態
  status          TEXT NOT NULL DEFAULT 'PENDING',
                   -- 'PENDING' | 'IN_PROGRESS' | 'DONE' | 'SKIPPED'

  completed_at    TIMESTAMPTZ NULL,

  -- 來源（給將來 AI / 分析用）
  source          TEXT NOT NULL DEFAULT 'manual',
                   -- 'manual' | 'ai_suggestion' | 'teacher_template'

  metadata        JSONB NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_study_tasks_plan_date ON study_tasks(plan_id, task_date);
CREATE INDEX idx_study_tasks_student_date ON study_tasks(tenant_id, task_date);
```

### 3. Optional: study_task_logs

若未來要紀錄學生實際花費時間 / 多次完成狀態，可加一張 log 表；
第一版可以先不實作。

## REST API (Draft)

Base path 建議類似：`/api/tenants/:tenantId` 之下。

### 1. 取得學生目前有效的讀書計畫

**GET** `/api/tenants/:tenantId/students/:studentId/study-plan`

回傳該學生在 `status = 'ACTIVE'` 的主要計畫及其任務。

**Response 200**

```json
{
  "plan": {
    "id": "...",
    "title": "學測前四週衝刺計畫",
    "targetExamDate": "2026-01-15",
    "startDate": "2025-12-18",
    "endDate": "2026-01-14",
    "plannedDays": 7,
    "plannedTotalHours": 21.0,
    "status": "ACTIVE",
    "config": {
      "weeklyDays": 3,
      "dailyHours": 1.5,
      "subject": "python"
    }
  },
  "tasks": [
    {
      "id": "...",
      "taskDate": "2025-12-18",
      "subjectCode": "python",
      "title": "複習第 1 章 + 習題 1-1 ~ 1-5",
      "durationHours": 1.5,
      "status": "DONE"
    },
    {
      "id": "...",
      "taskDate": "2025-12-19",
      "subjectCode": "python",
      "title": "複習第 2 章",
      "durationHours": 1.5,
      "status": "PENDING"
    }
  ]
}
```

若沒有 ACTIVE 計畫，回 404 或 `{ "plan": null, "tasks": [] }`。

---

### 2. 建立 / 覆蓋一份新的讀書計畫

**POST** `/api/tenants/:tenantId/students/:studentId/study-plan`

用於：
- 學生在前端按「產生本週讀書計畫」
- 老師 / AI 幫學生產生計畫

**Request body（範例）**

```json
{
  "title": "本週 Python 讀書計畫",
  "targetExamDate": "2026-01-15",
  "startDate": "2025-12-18",
  "endDate": "2025-12-24",
  "weeklyDays": 3,
  "dailyHours": 1.5,
  "subject": "python",
  "createdByRole": "student",
  "createdById": "...optional user id...",
  "tasks": [
    {
      "taskDate": "2025-12-18",
      "subjectCode": "python",
      "title": "複習第 1 章 + 習題 1-1 ~ 1-5",
      "durationHours": 1.5
    },
    {
      "taskDate": "2025-12-20",
      "subjectCode": "python",
      "title": "複習第 2 章",
      "durationHours": 1.5
    }
  ]
}
```

**行為建議**
- 將舊的 ACTIVE 計畫標記為 `ARCHIVED` 或 `CANCELLED`
- 新建一筆 `study_plans` + 對應多筆 `study_tasks`

**Response 201**

```json
{
  "planId": "...",
  "createdTasks": 5
}
```

---

### 3. 更新單一任務狀態（打勾完成）

**PATCH** `/api/tenants/:tenantId/study-tasks/:taskId`

**Request body 範例**

```json
{
  "status": "DONE"
}
```

也可以允許更新 `durationHours`、`title` 等欄位：

```json
{
  "status": "DONE",
  "durationHours": 2.0
}
```

**Response 200**

```json
{
  "id": "...",
  "status": "DONE",
  "completedAt": "2025-12-18T13:45:00Z"
}
```

---

### 4. 清除目前計畫（前端「清除本週計畫」對應後端版）

**DELETE** `/api/tenants/:tenantId/students/:studentId/study-plan`

行為：
- 將 ACTIVE 計畫標記為 `CANCELLED` 或 `ARCHIVED`
- 可選擇是否同步刪除對應 `study_tasks`

**Response 204**（無內容）

---

### 5. 老師 / 管理端：查詢學生計畫摘要列表

**GET** `/api/tenants/:tenantId/courses/:courseId/study-plan-summaries`

給老師看某門課的讀書計畫概況。

**Response 200（範例）**

```json
{
  "courseId": "c1",
  "items": [
    {
      "studentId": "...",
      "studentName": "王小明",
      "hasActivePlan": true,
      "currentWeekTotalPlannedHours": 4.5,
      "currentWeekCompletedHours": 3.0
    },
    {
      "studentId": "...",
      "studentName": "林小美",
      "hasActivePlan": false,
      "currentWeekTotalPlannedHours": 0,
      "currentWeekCompletedHours": 0
    }
  ]
}
```

## 實作建議（第一版）

1. **先做最小可用功能**
   - 實作 `study_plans` / `study_tasks` 兩張表
   - 實作：
     - `GET /students/:id/study-plan`
     - `POST /students/:id/study-plan`
     - `PATCH /study-tasks/:taskId`
     - `DELETE /students/:id/study-plan`

2. **前端接法**
   - 目前前端 StudyCoachView 使用 localStorage，可逐步改成：
     - 產生計畫 → call `POST /students/:id/study-plan`
     - 載入頁面 → call `GET /students/:id/study-plan`
     - 勾選任務 → call `PATCH /study-tasks/:taskId`
     - 清除計畫 → call `DELETE /students/:id/study-plan`

3. **權限與驗證**
   - 根據登入角色：
     - 學生只能讀寫自己的 `student_id`
     - 老師可針對授課班級的學生建立 / 查看計畫
     - 管理員可跨學生查看

這份 Spec 可以當成未來實作 backend（Node.js / Python 任一框架）的藍圖。
